Maven

可选依赖：对外隐藏当前资源对应的依赖关系---不透明​
排除依赖：隐藏其他资源中对应的依赖---不需要​
聚合工程：用来管理其他工程的一个根工程，此工程是一个独立的模块，内容只有一个pom文件，其中包含了packaging打包方式为pom,并设置了当前聚合工程所含有的子模块名称modules​
继承：子工程中可以继承父工程中的配置信息，并且减少版本之间的冲突​
1.在父工程中配置对应子工程中需要的依赖，并配置子工程中可选择的依赖，通过dependencyMannagement来设置
2.在子工程中配置继承的父工程通过parent来设置，relativepath填写父工程的pom文件位置
3.子工程中就可以使用父工程中的依赖，可以不指定版本
属性：类似于javaSE中的变量，通过properties来设置变量名，每个变量名代表版本信息，用到地方采用${变量名}即可,资源文件的引用也可以通过设置变量名，然后在不同的项目中的propertise文件通过${}来引用，但是引用源需要添加目录加载属性和资源过滤器​
版本：SNAPSHOT:快照版本，不断迭代，还没趋于稳定​
RELEASE:发布版本，比较稳定的版本 对外发布，还有诸多的例如alpha,beta,纯数字版​

多环境的开发：便于版本的切换（开发，测试，上线），通过添加profile,指定id为版本名称，propertis指定参数的值，然后通过activation标签表示默认环境或者通过mvn命令中 mvn 指令 -P 环境定义id 类:mvn install -P pro_env​

私服(组员之间共享)​

  • 配置本地仓库连接私服的密码
# 在maven的配置文件中添加server，可以设置多个
<server>
<id>自己建造的私服仓库名称<id>
<username></username>
<password></password>
</server>
  • 配置当前工程存放在私服的位置(一般release版本和snapShop版本分开存放)
#工程的pom文件中,通过deploy存放在对应的仓库当中
<distributionManahement>
<repository>
<id>仓库的名称</id>
<url>仓库的地址</url
<repository>
</distributionManahement>
  • 配置下载的地址（仓库组的地址
#在maven配置文件中
<mirror>
<id> 配置要本地仓库要访问的私服仓库组的名称</id>
<url>该仓库组的地址</url>
</mirror>）
