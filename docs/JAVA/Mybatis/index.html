Mybatis

JDBC​

  • -> sun公司定义的一套操作所有关系型数据库的规则，即各个数据库厂商区实现这套接口，提供数据库驱动的jar包，我们可以通过这套接口jdbc编程，真正执行的代码是驱动中的实现类
 DriverManeger:注册驱动，com.mysql.jdbc.Driver，获取连接getConnection

Connection:获取执行的sql对象（三种方法）,事务处理（回滚，开启，提交）

Statement:执行sql语句，执行sql对象.executeUpdate(sql),对象.executeQuery(),用来执行DQL语句，结果封装在resultSet结果集中，需要通过该集合中的某些方法获得对应的数据.
preparedStatement:防止sql注入，解决stateMent中存在sql注入问题，通过useServerPrepStmts=true开启,有预编译的功能

数据库连接池：是个容器负责分配管理数据库连接，提前申请连接，资源重用,导入durid jar包，定义配置文件，加载配置文件，获取连接池对象
再通过连接池对象获取数据库连接

使用Mapper代理

1.定义与sql映射文件同名的Mapper接口，并且将Mapper接口和SQL映射文件放置在同一目录下 2.设置sql映射文件的namespace属性为Mapper接口的全限定名 3.在Mapper接口中定义方法，方法名就是sql映射文件中的sql语句的id，并保持参数类型和返回值类型一致 4.通过sqlSession的getMapper方法获取Mapper接口的代理对象，调用对应的方法完成sql语句的执行

细节：如果Mapper接口名称和sql映射文件名称相同，并且在同一目录下，则可以使用包扫描的方式简化sql映射文件的加载

Mybaties的快速入门​

  1. 接收用户传入的参数，获取sqlSessionFactory

  2. 获取sqlSession对象

  3. 获取Mapper接口的代理对象

  4. 执行select方法，获取数据

  5. 释放资源

通过编写与Mapper接口相同的namespace，当调用Mapper接口中的方法，会先根据Mapper接口的全类名去找到映射文件，然后根据其中的方法找到对应的sql语句，映射文件中的id要和Mapper接口中的方法一致,xml文件类部记载着要查询的sql语句

接口的编写​

参数占位符：#{},将其替换为'?' ${} 拼sql，存在sql注入问题

特殊字符处理：当sql中运用<时，需要通过xml语法中的转义字符，或者CDATA区

动态sql语法

通过@parm()来替换系统默认参数值

接口的方法中的参数有多种形式，可以通过散装参数的方式来接收，或者封装成对象通过对象的方式来接收，或者通过封装成Map中的key

namespace :命名空间，相对于java文件夹下的接口名，这样在resultType中就可以直接使用预先设定的返回类型，不区分大小写

写sql语句时，如果出现的实体名和表中的变量名不同则可通过取别名的方式来消除错误，可以通过sql片段的方法来生成sq语句，然后sql语句中通过include 标签配合refid来代替即可.但是上面两者都存在局限性，最佳还是通过resultMap来实现映射，id主键映射，result普通键映射.

TOMCAT​

轻量级web服务器，支持servlet/jsp和少量javaEE规范,也称web容器，servlet容器 IDEA中创建maven-web的项目可以通过创建模块时的骨架 create from archeType。若不使用骨架，则需要在pom中添加war的打包方式，并补齐webapp项目文件 部署Tomcat: 1.集成Tomcat到项目中通过配置idea中的configuration

2.通过添加Tomcat插件，然后在pom中添加相应坐标即可

Servlet​

  • 用来开发动态web资源开发技术
  • 快速入门：导入servlet依赖坐标，并设置scope为provided,创建一个类实现server接口，并重写其中所有的抽象方法，在类上使用@webServlet注解配置Servlet的访问路径。
  • servlet的生命周期：tomcat创建servlet对象，并且会主动调用我们实现接口类中的方法，其中第一次访问时会调用init方法（一次），多次调用servlet方法.
  • HttpServlet:是servlet的一个实现类，底层对一些请求方法进行了封装，使得调用者可以通过不同的调用方法处理某个请求

  • Servlet urlPath:一个servlet可以配置多个urlPath

  • 如何解决通过request请求获取的数据出现乱码的问题，post中通过设置setCharcterEncoding(UTF-8),get请求通过URLEncoder和URLDecoder来实现编码和解码的过程

  • 请求转发：req.getRequestDispather("资源B路径").forward(req,res)

代理​

JDK动态代理：通过底层的反射机制来自动生成java类，要求目标类与代理类实现相同的接口​
GGLIB动态代理：(Code Generation Libary) 开源项目，功能强大，代理效率高于jdk,高性能高质量的Code生成类库，可以在运行期扩展java类与实现java接口，广泛的被许多AOP的框架使用，代理的原理是生成目标类的子类而字类是增强的，子类对象就是代理对象，所以要求目标类必须能够被继承不能是final类​
