springboot

propertise>yml>yaml文件的执行顺序，其中yml,yaml填写数据时需要加空格​

YAML数据读取的方式​

单个数据：@value()，属性名引用方式：${一级属性名，二级属性名....}​
全数据读入：封装成Environment对象，通过env.getProperty("id")获取，框架内部用的较多，或者自定义一个对象来封装数据，将该类交给spring框架，并写入注解@ConfigurationProperties(prefix="")​
多环境开发，yml文件用---来隔开，不同环境通过profile来表示，配置环境通过active来选择,propertise文件通过spring.propertise.active=id来实现，不同的环境要生成不同的文件，不同的文件用这种application-pro.propertise格式来表示​
带参数启动spring boot ：java -jar springboot.jar --spring.profiles.active=test​
配置文件​
springBoot中四级配置文件
1级：file :config/application.yml     优先级最高
二级：file:application.yml
三级:classpath:config/application.yml
四级：classpath:application.yml            优先级最低

整合mybatisplus​

定义domain实体类，定义数据层接口，使其继承BaseMapper<实体类名>,并添加注解@Mapper ，即可在测试类中添加查找语句​

条件查询​

  • 设置查询条件

    lambda 格式

QueryWrapper<User> qw=QueryWrapper<User>();
qw.lambda().lt(User::getAge,65).ge(User::getAge,18)
List<User> userList=userDao.selectList(qw);
System.out.println(userList);
  • 查询投影 （结果包含模型类中未定义的属性）
QueryWrapper<User> qm=new QueryWrapper<User>();
qm.select("count(*) as nums,gender");
qm.groupBy("gender");
List<Map<String,Object>> maps=userDao.selectMaps(qm);
System.out.println(userList);
字段射和表名映射关系可以通过@TabFild()和@TableName来修改​
