Spring

深入了解框架的使用之前，先给你介绍一下广泛用于其中的概念

IOC (Inversion of Control)：控制反转,对象的创建由内部转向外部，IoC容器负责对象的创建，初始化一些列工作，被创建或者被管理的对象在IOC容器中统称为Bean.

DI（Dependency insert）:在IOC容器中如果需要建立bean于bean之间的关系，则称之为依赖注入。

通过xml文件的格式配置：通过在需要注入依赖的实现类中配置setter方法，并在xml文件中配置propoty属性通过name连接set方法，通过ref连接当前xml文件中的bean id（注入源）.

通过注解的方式配置：直接通过 @AutoWired注入即可

    • BeanFactory:Spring的早期接口，称为spring的Bean工厂，更偏向与底层，首次调用getBean时才进行Bean的创建.
    • ApplicationContext:在BeanFactory基础功能上进行了扩展，例如监听功能，国际化功能，内部维护了BeanFactory，随着配置文件的加载，容器一创建就将Bean实例化并且初始化.

bean的配置​

bean中的id属性在底层会封装成bean name,然后测试方法中其实是通过的bean name来获取bean对象的，当没有id时也可以通过class权限名来获取bean,name是别名，可以起多个别名，别名也指向id。

  • scope
    singleton:单例默认值，随着类的加载创建，并存储到spring容器的内部singletonObject池中，每次getbean时从中获取.
    prototype:调用getBean时才会创建，每次创建出来的bean地址都不一样,不会放到单例池中.

  • lazy-init:延迟加载，getBean时才会创建实例，

  • Init-method属性和destory-method属性配置，前者在实例后执行，后者在容器销关闭前执行.还有一种方法就是实现initializingBean的接口，其中有一个方法名为afterPropertiesSet方法，也可以进行一些初始化操作，顺序是在属性设置完毕之后，在init之前.

  • bean的实例化配置：默认为无参的构造方法，如果实例中提供的方法为有参构造的话则需要配置相应的constructor-arg来指定参数。还可以通过静态工厂的方式来配置实例化对象，创建一个class对象后添加另一个对象为返回值的方法，通过factory-method指定该方法即可。实例化工厂的方式配置对象，先产生对象，然后再指定对象其中的方法。factoryBean延迟实例化bean的方法底层是先产生一个单例池，单例池中存储了该实例化工厂，当调用getbean时才会从工厂的缓存区去取得实例化对象

    手动注入list:需要通过在propertise中配置list标签然后添加value或者bean的方式设置值

    自动装配：通过添加autowrite属性，spring会自动在该实现类中自动寻找set方法来进行装(注入),该属性有两个值bytype，byname

  • beans标签：可以通过profile设置多套环境，然后在测试代码中可以通过命令行动态设置参数或者通过代码System.SetProperty("spring.profiles.active","新配置的环境名")

  • import标签：可以引入其他模块的配置文件

  • alias标签：取别名,存放在aliasMap中

Bean 实例化的基本流程​

bean标签的封装，包含了bean标签中的所有内容，最后定义为bean defination 对象，并存储在bean defination Map集合中 ，applicationContex底层遍历bean defination Map创建bean 对象，并存储到singletonObjects的Map中，当执行getBean时从当地池中获取即可

spring 后处理器（beanFactory后处理器，bean后处理器）

BeanFactoryPostProcessor:(放入defination Map之前)实现该接口，配置到容器中即可使用,可以在该阶段修改bean definattion，子接口BeanDefinationRegistryPostProcessor专门用于注册Bean defination操作

注解思维：简单理解底层通过扫描当前项目中所有类里使用注解了的类名，将其通过后处理器的方式注册为bean defination 对象，最后将这个注册的方法交给spring管理即可

BeanPostProcessor:(放入 singletonObjects 之前,bean实例化之后)可以对bean做一些修改在存入之前

属性注入​

普通属性或者单向注入时，是通过set方法反射进去的，如果容器中没有，则先创建被注入的对象实例，再进行注入操作

循环注入：采用的是三级缓存结构，bean a依赖于bean b，而bean b也依赖于bean a

singletonObjects :存储的是一级缓存，及实例化和初始化完成后的bean map<String,Object>

earlySingletonObjects:二级缓存，缓存半成品对象，且当前对象已经被其他对象引用 map<String,Object>

singletonFactories:三级缓存，缓存半成品对象，使用时再通过工厂创建Bean，未引用 map<String,ObjectFactory<?>>

Aware​

Aware接口是一种框架辅助属性注入的一种思想，其他框架中也可以看到类似的接口，框架具有高度封装性，我们接触到的一般都是业务代码，一个底层功能API不能轻易的获取到，但是这不意味着永远用不到这些对象，如果用到了就可以使用框架提供的Aware接口，让框架给我们注入对象.

AOP​

ASpect Oriented Programing 面向切面编程，在不惊动原始设计的基础上为其进行功能的增加（无侵入式），本质：代理模式

通知类中的方法称为通知方法，每个可以使用通知类中的方法（称通知，共性功能 ）的点叫做连接点，而与通知连接的点又称为切入点，而切入点通知之间的关系称为切面

@Component  //加入到spring容器
@Aspect //表示该类使用的是面向切面
public class Myadvice{
    @Pointcut("execution(void com.itheima.dao.BookDao.update())")//定义切入点和设置切入点的方法
    private void pt(){}
    @Before("pt()")     //执行时计
    public void before(){
    system.out.println(System.currentTimeMillis())
    }
}
最后在spring的核心配置中开启@EnableAspectJAutonProxy即可完成Aop

切入点表达式：修饰符 返回值 pathReference 方法（参数），可以使用*(任意符号)和..(多个任意符号)和+（匹配子类型）来简化代码

exuction public com.itheima. . Service .find (*) 匹配该包下任意包中的UserService类或接口的所有以find开头的带有一个任意参数的方法

AOP的通知类型有五种：前置通知（before）获取形参joinpoint，后置通知（after），*环绕通知（Around）需要依赖形参proceddingJoinPoint才能实现对原始方法的调用(pjp.proceed)，方法返回值设置为object，方法结束(afterReturring)，方法抛异常(afterThrowind)
